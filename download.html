Stable Releases

The disk images countain source code and binaries. The binaries are built with the SDKs corresponding to the highest version numbers listed in the platforms field. They are built for deployment on systems that run at least the lowest version listed. It is usually possible to build a given version of OCMock from source for older versions of OS X or iOS, at the expense of losing some features.

Source code

The source code for OCMock is available on GitHub

  https://github.com/erikdoe/ocmock

It is built and tests are run by Travis CI. The current status is shown above on the right. If this isn't showing passing, chances are the code is currently not building correctly and we're busy trying to fix it. In all other cases, a local build from source should work.

The easiest way to build OCMock from source is to use the supplied build script. 

cd Tools
ruby build.rb

Running this script should create binary versions of the OCMock framework and library in a directory under /tmp. When the build is complete a finder window for the directory should open up automatically.

If you pass the -r flag to the script it creates a disk image with all the artefacts in it. This is what we use to create the binary distribution.

If you have switched compilers using the CC enviromnent variable the build might fail. In such cases, please reset the variable before running the script.
